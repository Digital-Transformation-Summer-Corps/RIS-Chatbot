Architecture Overview:

This diagram shows the architecture of a Retrieval-Augmented Generation (RAG) chatbot system that processes documentation from Confluence and provides intelligent responses to user queries.

  • Data Sources (Blue): Confluence documentation that serves as the knowledge base
  • Processing (Purple): Data scraping and text processing components
  • Storage (Green): Vector database for semantic search and retrieval
  • Interface (Orange): Web server and user interaction components
  • AI Models (Pink): Embedding models, Large Language Models, and reasoning models
  • Validation (Teal): Quality assurance and testing framework

Data Flow:

  1. Documentation is scraped from Confluence
  2. Text is processed and converted to vector embeddings
  3. Embeddings are stored in a vector database
  4. User queries are processed and matched against stored vectors
  5. Relevant context is retrieved and sent to the LLM
  6. The LLM generates responses based on the context
  7. Validation systems ensure quality and accuracy
